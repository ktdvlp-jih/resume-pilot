# ResumePilot 관리자 가이드

> 대상: 관리자(`ADMIN`) 계정  
> 기준: 2026-07-07 배포 상태

## 1) 접속과 권한

- 관리자 화면 경로: `/admin/`
- 일반 사용자 계정은 관리자 메뉴 접근 불가
- 운영 점검 전, 관리자 계정으로 재로그인해 세션을 갱신

## 2) 관리자 핵심 메뉴

### 2-1. Prompts

용도:

- 프롬프트 템플릿/버전 관리
- 활성 버전 전환
- 버전 간 diff 비교

운영 체크:

- 활성 버전이 의도한 버전인지 확인
- 변경 전/후 diff 확인 후 저장
- 프롬프트 변수(`{{...}}`) 누락 여부 확인

### 2-2. Forbidden Expressions

용도:

- 생성 결과에서 피해야 할 표현 정책 관리

운영 체크:

- 금지어 추가/수정 후 생성 품질 회귀 확인
- 과도한 금지어로 생성 품질 저하가 없는지 확인

### 2-3. AI Logs

용도:

- AI 호출 이력 모니터링 (`ai_usage_logs`)

운영 체크:

- `generate`, `detect`, `review`, `interview`, `compare_keywords` 호출 여부
- 오류/지연 증가 추세 확인

### 2-4. Users / Companies

용도:

- 사용자/회사 데이터 확인

운영 체크:

- 신규 사용자 생성 흐름 이상 유무
- 공고 분석 후 회사 정보 반영 여부

## 3) 프롬프트 변경 운영 절차 (권장)

1. 변경 목적 정의 (무엇을 개선할지 1문장)
2. 초안 수정 및 버전 저장
3. 버전 diff로 변경점 검토
4. 스테이징 사용자 시나리오로 생성 테스트
5. 운영 반영 후 AI Logs 추적

## 4) 장애 대응 빠른 체크리스트

1. `/admin/ai-logs`에서 최근 실패 요청 확인
2. 서버에서 `docker compose ps` 상태 확인
3. `deploy-smoke.sh`로 HTTP/API 기본 상태 확인
4. 생성 관련이면 다음 순서로 점검:
   - `resume-ai` 상태
   - `prompt-service` 상태
   - `rag-service` 상태
   - `OPENAI_API_KEY`/`OPENAI_BASE_URL` 설정
5. 필요 시 최근 프롬프트 변경 롤백

## 5) 운영 보안 주의사항

- `.env` 실값(토큰/비밀번호/API 키)은 Git/채팅/문서에 노출 금지
- 운영 DB 수동 수정 시 변경 사유 기록
- 관리자 계정 비밀번호 재사용 금지

## 6) 최소 일일 운영 루틴 (5~10분)

1. `AI Logs` 최근 호출/실패 확인
2. 주요 프롬프트 활성 버전 확인
3. 공고 분석 1건 + 생성 1건 샘플 체크
4. 이슈 발생 시 `docs/reports/`에 날짜별 리포트 추가
