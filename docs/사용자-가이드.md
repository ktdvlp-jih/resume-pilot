# ResumePilot 사용자 가이드

> 대상: 일반 사용자 (web)
> 기준: 2026-07-08 배포 상태

## 1) 시작하기

1. 서비스 접속 후 회원가입/로그인
2. 대시보드에서 전체 진행 상태 확인
3. 좌측 메뉴 순서대로 진행: `경험 라이브러리 → 채용공고 → 워크스페이스`

## 2) 경험 라이브러리 작성

핵심 원칙은 **구체성**입니다.

- 제목: 프로젝트/업무명을 명확히 작성
- 설명: 무엇을 만들었는지, 어떤 문제를 해결했는지 작성
- 역할: 본인의 책임 범위를 구체적으로 작성
- 결과: 수치(%, 시간, 비용, 건수) 중심으로 작성

권장: 2~5개 경험을 먼저 등록하면 RAG 추천 정확도가 올라갑니다.

## 3) 채용공고 업로드/분석

`채용공고` 메뉴에서 3가지 방식 지원:

- 텍스트 붙여넣기
- URL 입력
- 파일 업로드(PDF/PNG/JPG 등)

분석 완료 후 확인할 항목:

- 회사명/포지션
- 필수/우대 기술
- 기술 키워드
- 핵심 역량

주의: 이미지 공고는 OCR 품질에 따라 회사명 인식이 다를 수 있습니다. (저해상도면 텍스트/URL 업로드 권장)

## 4) 워크스페이스에서 자소서 생성

1. 채용공고 선택 또는 공고 텍스트 입력
2. `관련 경험 추천` 실행
3. Rewrite 레벨(0~100) 조절
4. `자기소개서 생성` 실행

결과 패널에서 확인:

- 추천 경험(RAG)
- 품질 점수
- 키워드 매칭/누락
- AI 흔적 탐지
- 첨삭/예상 면접 질문

## 5) 품질 높이는 팁

- 경험의 결과를 수치화할수록 생성 품질이 올라갑니다.
- 공고 원문의 필수 기술을 누락 없이 입력하세요.
- 생성 결과는 초안으로 보고, 본인 경험 사실관계 검토 후 확정하세요.

## 6) 자주 묻는 문제

- **추천 경험이 비어 있음**: 경험이 너무 적거나 키워드가 약함 → 경험 2개 이상 보강
- **AI 생성 실패**: 잠시 후 재시도, 공고 텍스트 길이 축소, 네트워크 상태 확인  
  (운영에서는 다중 Provider failover가 자동 적용되지만, 모든 Provider 한도 소진 시 실패할 수 있음)
- **분석 품질이 낮음(이미지 공고)**: 해상도 높은 이미지 또는 텍스트/URL 방식 권장

## 7) 개인정보/보안 주의

- 이력서 원문/민감정보는 필요 최소한만 입력
- 비밀번호 재사용 금지
- 공용 PC 사용 후 반드시 로그아웃

## 8) 추천 사용 순서 (10분 온보딩)

1. 경험 2~3개 등록 (3분)
2. 채용공고 업로드/분석 (2분)
3. 워크스페이스 추천+생성 (3분)
4. 탐지/첨삭 반영 후 저장 (2분)
